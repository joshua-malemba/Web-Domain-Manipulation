Homework 1: Tasks for jm850

Your personal task is to develop a system of TV channels and shows running on these channels.

Task 1: Configuration [4 marks]

Configure the project to support the use of JSPs as view technology. JSPs must be configured to be placed inside folder src/main/webapp/WEB-INF/views/.

Task 2: Domain Classes [13 marks]

Create class co2103.hw1.domain.Show. With the following private fields:

  1. String title
  2. String producer
  3. String category
  4. int episodes
Generate necessary getters and setters. [7 marks]

Create class co2103.hw1.domain.Channel. With the following private fields:

  1. int id
  2. String name
  3. String topic
  4. List<Show> shows
Generate necessary getters and setters. [6 marks]

Task 3: Controller (simple) [5 marks]

Create controller co2103.hw1.controller.MainController [2 marks].

  1. Requests for URL "/" (POST and GET) should show the view start (the corresponding JSP already exists). [3 marks]

Task 4: Controller, View, and Form [20 marks]

Create field public static List<Channel> channels in class co2103.hw1.Hw1Application. Initialize the list inside class co2103.hw1.Hw1Application to contain one channel with two shows. [4 marks]

Create controller co2103.hw1.controller.ChannelController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/channels" should show view channels/list. [2 marks]
    • Add the list of all channels from co2103.hw1.Hw1Application to the model as attribute channels. [2 marks]
    • Create the corresponding JSP to list the name and topic of each channel followed by a link to URL "/shows?channel={id of channel}", e.g., "/shows?channel=3". [4 marks]
  2. Requests for URL "/newChannel" should show view channels/form. [3 marks] Create the corresponding JSP with a form with modelAttribute="channel" and action="/addChannel" and inputs for all fields of class Channel except for shows. [5 marks]
  3. POST requests for URL "/addChannel" should add a new channel to the list of all channels in co2103.hw1.Hw1Application and redirect to /channels. [3 marks]

Task 5: Controller, View, and Form with additional Parameter [33 marks]

Create controller co2103.hw1.controller.ShowController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/shows" should show view shows/list. [2 marks]
    • Based on request parameter channel determine the corresponding channel from co2103.hw1.Hw1Application. [2 marks]
    • Add the list of all shows of that channel to the model as attribute shows. [2 marks]
    • Create the corresponding JSP for the view to list the title, producer, and category of each show. [4 marks]
    • At the end of the list in the JSP add a link to URL "/newShow?channel={id of channel whose shows are listed}", e.g., "/newShow?channel=3". [4 marks]
  2. Requests for URL "/newShow" should show view shows/form. [3 marks]
    • Create the corresponding JSP with a form with modelAttribute="show" and action="/addShow" and inputs for all fields of class Show. [6 marks]
    • Also add the request parameter channel with the ID of the current channel. [4 marks]
  3. POST requests for URL "/addShow" should add a new show to the list of shows of the channel identified by request parameter channel and then redirect to /channels. [6 marks]

Task 6: Validation [25 marks]

Set up a validator for class Channel and make it available in controller ChannelController. [4 marks]

When adding a new channel validate that

  1. The value of field id is unique within the list Hw1Application.channels. [3 marks]
  2. The value of field name is not empty. [2 marks]
  3. The value of field topic is not empty. [2 marks]
In case of validation errors go back to the form and show error messages for each field in the form.

Set up a validator for class Show and make it available in controller ShowController. [4 marks]

When adding a new show validate that

  1. The value of field title is not empty. [2 marks]
  2. The value of field producer is not empty. [2 marks]
  3. The value of field category is either "Comedy", "Documentary", or "Drama". [3 marks]
  4. The value of field episodes is between (including) 1 and 24. [3 marks]
In case of validation errors go back to the form and show error messages for each field in the form.
